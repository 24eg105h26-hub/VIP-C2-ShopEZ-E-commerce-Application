2. Project Overview
• Purpose
ShopEZ is developed to address key operational challenges in the modern e-commerce
landscape. For small and medium-sized merchants, it provides a highly customizable and
scalable online storefront to showcase products, manage inventory, and track orders. For
end-consumers, it offers a fast, intuitive, and secure shopping experience with dynamic
catalog filtering, persistent state retention, and multi-gateway payment processing.

• Features
User Management & Authentication: JWT sessions stored in secure HTTP-only
cookies, combined with BCrypt password hashing and role-based access checks
(Admin vs. Customer).
Dynamic Product Catalog: Pagination, keyword search, and filters (by category,
price range, and rating) working without full-page reloads.
Persistent Shopping Cart: Syncs shopping cart items directly with MongoDB to
retain user selections across multiple devices.
Secure Checkout Flow: A multi-step checkout wizard integrated with
Stripe/Razorpay for secure card and UPI payment processing.
Admin Management Panel: Full CRUD operations for products, sales metrics
charts, and status tracking for order fulfillment.
Media Handling: Ephemeral uploads optimized and served via the Cloudinary
CDN.
3. Architecture
• Frontend
The client is structured as a single-page application (SPA) using React.js. It features:
Context API: Handles global state for user authentication sessions and shopping
cart items.
Axios Interceptors: Manages REST API communication, injecting credentials
automatically and centralizing error notifications.
Responsive Styling: CSS variables and HSL palettes optimized for mobile, tablet,
and desktop viewports.
• Backend
The server runtime is powered by Node.js and structured using Express.js in a strict
Model-View-Controller (MVC) architecture:
Routing: Decoupled endpoints for users, products, and orders.

Middleware: Intercepts requests for authentication (protect), role checking
(isAdmin), and global exception handling (errorMiddleware).
• Database
We utilize MongoDB Atlas as our cloud NoSQL data store, modeled and queried using
the Mongoose ODM:
Data Integrity: Enforces strict schemas and validates inputs before writing to the
collections.
Referential Constraints: Uses Mongoose ObjectId references to connect users,
carts, products, reviews, and orders.

4. Setup Instructions
• Prerequisites
Node.js: v18 LTS or above installed.
MongoDB: A cloud cluster on MongoDB Atlas or local installation (v6.x+).
Package Manager: npm v9.x or above.
External accounts: Stripe account (for API test keys) and Cloudinary account (for
media uploads).
• Installation
1. Configure Environment Variables:
Create a .env file under the /server folder with the following variables:
PORT=8000
MONGO_URI=mongodb+srv://<username/:<password>@<cluster>.mo
JWT_SECRET=your_jwt_secret_key
STRIPE_SECRET_KEY=sk_test_your_stripe_key
CLOUDINARY_CLOUD_NAME=your_cloudinary_cloud_name
CLOUDINARY_API_KEY=your_cloudinary_api_key
CLOUDINARY_API_SECRET=your_cloudinary_api_secret
2. Install Dependencies & Seed Database:
# Install server dependencies & seed DB
cd server /& npm install /& npm run seed
# Install client dependencies
cd /./client /& npm install

8. Authentication
Authentication on the ShopEZ platform is stateless and utilizes JSON Web Tokens
(JWT).
Token Issuance: When a user registers or logs in, the server generates a token signed
with the JWT_SECRET.
Cookie Storage: The token is written directly to the response header as an HTTP-
only, SameSite=Strict cookie. This prevents cross-site scripting (XSS) extraction
and cross-site request forgery (CSRF) vulnerabilities.
Authorization: Protected routes use the JWT validation middleware:

10. Testing
The testing strategy for ShopEZ ensures reliability and correctness across multiple levels:
Unit Testing: Testing of utilities, validation rules, and context status reducers (Jest,
RTL).
Integration Testing: Running API request/response assertions using Postman and
Newman collections to verify route security and status codes.
User Acceptance Testing (UAT): Execution of 16 defined manual UAT scripts
covering complete user flow (Registration → Search → Cart → Checkout →
Payment → Admin Inventory verification).
